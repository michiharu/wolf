Tree ¶

マニュアルツリー  ¶

DB未保存のデータはidを"tmp:~"で始めるルールとする。 おそらく親となるレコードのIDを子が持つDB設計になると思うが、 レコードの削除はrootとなるマニュアルデータ以外は明示的なAPIを持たないため、 編集時に削除されたデータがあるかどうかをチェックし、 あれば該当レコードを削除するロジックをサーバーサイドで組む必要がある。
